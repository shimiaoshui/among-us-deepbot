# Among Us DeepBot 完整使用教程

## 1. 先选对下载包

原版房间使用 `Standalone` 包。已经使用 The Other Roles v4.6.0 汉化版的房间，房主和每一位真人玩家都使用 `TOR46-Compatibility` 包，并保证游戏、BepInEx、Reactor、TOR 的版本完全一致。

兼容包只提供 DeepBot 兼容层，不重新分发 TOR。不要把两个不同版本的 TOR DLL 混在同一个房间。

## 2. 安装前准备

1. 在 Steam 中右键 Among Us，选择“管理 → 浏览本地文件”。
2. 备份整个游戏目录，或至少备份 `BepInEx\plugins` 与 `BepInEx\config`。
3. 安装适配当前 Among Us 版本的 BepInEx 6 IL2CPP，并启动一次游戏，让 `BepInEx\interop` 生成完成。
4. TOR 兼容版还要先确认 TOR v4.6.0 汉化版可以单独正常开局。

## 3. 安装 DeepBot

1. 解压所选压缩包到任意临时目录，不要直接在压缩包内运行。
2. 双击 `Install-DeepBot.cmd`。
3. 脚本会自动寻找常见 Steam 路径；找不到时输入包含 `Among Us.exe` 的游戏目录。
4. 安装器只复制 DeepBot DLL、无密钥配置模板和两个本地脚本，不覆盖游戏本体。

手动安装也可以：把 `BepInEx\plugins\AmongUsDeepSeekBots.dll` 复制到游戏的同名目录，把配置文件复制到 `BepInEx\config`。

## 4. 永久设置 API 密钥

只在房主电脑上运行游戏目录里的 `Configure-DeepBot-Key.cmd`，按提示输入自己的 Agnes/OpenAI 兼容 API 密钥。

密钥会保存到：

```text
%LOCALAPPDATA%\AmongUsDeepSeekBots\api-key.txt
```

它不在游戏目录，也不在插件配置文件中。以后用 `Start-DeepBot.cmd` 启动时不再重复询问。客端不需要 API 密钥。若想更换密钥，重新运行配置脚本；若想删除，删除上面的文件即可。

## 5. 原版本地/LAN 联机

1. 房主安装 `Standalone` 包并设置密钥。
2. 客端建议安装相同版本的 DeepBot，以保证 modded 客户端行为一致；把客端配置中的 `BotCount = 0`。
3. 所有人从游戏的“本地”模式进入同一个局域网房间。
4. 由房主决定 `BotCount`，默认 5。只有房主创建和驱动 Bot。
5. 房间里的击杀冷却、内鬼人数、任务数等设置保持最高优先级。

## 6. TOR 4.6.0 兼容联机

1. 所有人先安装同一份 TOR 4.6.0 汉化版，并确认不装 DeepBot 时能够联机开局。
2. 所有人再安装 `TOR46-Compatibility` 包。
3. 只有房主设置 `BotCount` 为需要的数量并配置 API 密钥；客端设置 `BotCount = 0`。
4. TOR 设置里选择的职业数量、概率、冷却、任务与胜利规则才是最终标准。DeepBot 读取并遵循它们，不在配置里另造一套规则。
5. 主职业只能有一个；恋人、诱饵、迷你船员、职业交换者等附加职业可以与主职业共存。

## 7. 常用配置

配置文件：`BepInEx\config\local.amongus.deepseekbots.cfg`

- `BotCount`：房主生成的 Bot 数；客端设为 0。
- `Model`：OpenAI 兼容模型名，默认 `agnes-2.0-flash`。
- `ApiBaseUrl`：默认 `https://apihub.agnes-ai.com/v1`。
- `MeetingUseDeepSeek`：会议讨论和投票是否调用模型。
- `BotUseRoleAbilities`：是否允许 Bot 使用职业技能。
- `PostMatchReflection`：是否让每个 Bot 在赛后写入去重经验。
- `SpeedMultiplier`：移动速度倍率，过高更容易在狭窄区域抖动。

不要在配置文件中添加 `ApiKey` 字段。

## 8. 如何确认安装成功

进入房间后检查 `BepInEx\LogOutput.log`：

- 应出现 `Among Us DeepSeek Bots 0.9.9-tor46-full-role-recognition loaded`。
- TOR 房间应出现 `customRoles=44, modifiers=11`。
- 每个 DeepBot 分配身份后应出现 `primaryRole=...` 与 `modifiers=...`。
- 日志不应出现 API 密钥正文。

## 9. 故障排查

无法开局：先把 `BepInEx\plugins` 中重复、旧版或 `.bak` DLL 移出插件目录，再确认所有真人玩家版本一致。

Bot 不动或抖动：保留日志中的 `DeepBot no route`、`stuck recovery`、当前位置和目标点；不要只截画面。The Skeld 以外地图目前不是本版本的完整路径覆盖范围。

会议不回复：确认房主密钥文件存在、接口余额/权限正常，并在日志中查找 HTTP 状态。接口失败时会退回本地逻辑。

职业技能不执行：确认 `BotUseRoleAbilities = true`，再核对 TOR 房间设置是否禁用了该职业/技能、次数是否用完、冷却是否结束、目标是否合法。

看到两种主职业：先区分黄色附加职业和真正主职业。若确实有两个主职业，请保存从身份分配到开局后的完整日志。

## 10. 卸载与恢复

删除 `BepInEx\plugins\AmongUsDeepSeekBots.dll` 即可停用。需要恢复旧版时，从自己的备份中还原 DLL 和配置。删除 `%LOCALAPPDATA%\AmongUsDeepSeekBots` 会同时清除密钥与 Bot 的持久化经验。
